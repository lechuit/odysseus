# Product

## Register

product

## Users

Odysseus is primarily for local-first power users, builders, and admins who want an AI workspace they can run and inspect themselves. They use it in long, tool-heavy sessions: coding, research, document work, shell automation, email triage, notes, tasks, and local model experiments.

## Product Purpose

Odysseus gives a self-hosted AI agent a safe, practical workspace: chat, tools, files, shell, documents, search, integrations, and local model workflows in one interface. Success means the agent can do real work while the user always understands what context is active, what operation is about to run, and where the safety boundary is.

## Brand Personality

Warm, technical, and trustworthy. The interface should feel like a capable local cockpit: direct, legible, a little playful, but never opaque or performative.

## Anti-references

Avoid cloud-first SaaS gloss, hidden automation, vague “AI magic,” noisy dashboards, and security flows that make users guess what is being allowed. Avoid memory or RAG behavior that silently mixes context across chats.

## Design Principles

- Make agency visible: show what the agent is doing, what it needs, and why.
- Keep local trust first: permission, sandbox, memory, and workspace boundaries should be explicit.
- Prefer calm density over decoration: users are doing work, not watching a landing page.
- Make sharp tools feel safe: dangerous operations need clear scope, copy, and reversible choices when possible.
- Preserve continuity: long chats should stay understandable without surprising context jumps.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve visible keyboard focus, minimum touch target sizes for interactive controls, clear reduced-motion behavior, and plain-language security copy that does not rely on color alone.
